GreenRoute implementation plan — one-day sprint, technical architecture, and API details

One-day sprint — 8 phases across ~10 hours

1

Setup & API keys

Hour 0–1
  • Register at datamall.lta.gov.sg → get LTA AccountKey (instant)
  • Register at www.onemap.gov.sg/apidocs → get OneMap token (instant)
  • npx create-next-app greenroute — App Router, Tailwind, TypeScript
  • Add .env.local: LTA_KEY, ONEMAP_TOKEN, ANTHROPIC_API_KEY
2

Route data layer

Hour 1–3
  • API Route /api/geocode — wraps OneMap Search to resolve place names → lat/lng
  • API Route /api/route — calls OneMap Public Transport routing (routeType=pt) for MRT/bus itineraries + OneMap walk routing (routeType=walk)
  • Hardcode carbon constants per mode (see table below)
  • Return unified payload: legs array, total distance, estimated time, CO₂ grams
4

UI — search & results

Hour 4–6
  • Origin/destination input with OneMap autocomplete (debounced, 300ms)
  • Route cards: MRT, Bus, Walk — each showing time, distance, CO₂ grams, and fare (from OneMap)
  • Visual CO₂ comparison bar per mode
  • Green badge on lowest-emission option
  • Claude AI nudge panel (streamed text below results)
5

Car/Grab baseline

Hour 6–7
  • Use OneMap drive routing (routeType=drive) for distance in km
  • Apply Singapore car emission factor: 147 gCO₂/km (LTA 2023 average)
  • Display "You'd save X grams vs driving" prominently — this is the hero metric
  • Optional: add Grab fare estimate via scrape or hardcoded rate (~$0.65/km)
6

Savings tracker

Hour 7–8
  • LocalStorage: accumulate CO₂ saved per trip
  • Dashboard widget: "Total CO₂ saved this session: X kg = Y trees for a day"
  • Share card: generate a shareable PNG snippet (html-to-image) for demo
7

Deploy

Hour 8–9
  • vercel --prod — add env vars in Vercel dashboard
  • Test 3 real routes: Jurong East → Raffles Place, Tampines → Orchard, Choa Chu Kang → Novena
  • Fix any CORS issues on LTA API (proxy all via Next.js API routes)

API reference

OneMap (SLA)

Geocode (place → coords)

GET /commonapi/search?searchVal={q}&returnGeom=Y&getAddrDetails=N

Public transport routing

GET /api/public/routingsvc/route?start={lat,lng}&end={lat,lng}&routeType=pt&token={token}

Returns walking + MRT/bus legs, total duration, fare in SGD

Drive routing (car baseline)

routeType=drive

Nearest MRT/bus stop

GET /api/public/nearbysvc/getNearestMrtStops?latitude={}&longitude={}&radius_in_meters=1000

LTA Datamall

Real-time bus arrival

GET /ltaodataservice/v3/BusArrival?BusStopCode={code}

Header: AccountKey: {LTA_KEY}

Train service alerts

GET /ltaodataservice/TrainServiceAlerts

Show disruption banner → nudges toward bus/walk

Auth

Free, instant via datamall.lta.gov.sg. AccountKey in header — never expose in client.

Risk flags & mitigations

LTA CORS block

LTA API must be called server-side. Never call from browser. All LTA calls go through /api/route Next.js route.

OneMap token expiry

Token expires in 3 days. Refresh via POST /api/auth/post/getToken with credentials. Cache in server memory for hackathon scope.

No walking + PT combo

OneMap PT route already includes walk legs. Don't double-count. Extract walk distance separately from each leg's mode: WALK field.
